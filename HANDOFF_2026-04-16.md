## Current Ratings Logic Notes

- Player matching should use season + player name first.
- `stats/manual/playerlist.csv` is the current player universe file.
- Playmaking currently keeps `playerlist.csv` as the identity universe, enriches `RotationRole` from
  `stats/history/bball_index_playmaking.csv` when available, keeps live `MIN` on the NBA.com side,
  and now reads `Speed with Ball -> SPEED` from `stats/exports/physical_all_audit.csv -> Speed AggregateZ`
  when that Physical audit export is present.
- Physical currently keeps `playerlist.csv` as the identity universe, enriches `RotationRole` from
  `stats/history/bball_index_physical.csv` when available, mirrors the workbook behavior where
  `Agility -> SPEED` consumes the Physical `Speed` score directly, and builds its first pass from:
  - a live bball-index physical export in `stats/history/bball_index_physical.csv`
  - NBA.com history in `general_traditional.csv`, `general_scoring.csv`, `general_advanced.csv`,
    `bios.csv`, `tracking_speed.csv`, `playtype_transition.csv`, `playtype_offscreen.csv`,
    `playtype_cut.csv`, and `hustle.csv`
  - combine data in `draft.csv` and dunk-tracking history in `overall_dunk_stats.csv`
  - current database attributes in `Upgrade/players.csv`
  - current cross-category score feeds from the finishing audits plus
    `stats/manual/2k26_rebound.xlsx -> Offensive Rebound` and `stats/manual/2k26_defense.xlsx -> Block`
- Impact currently keeps `playerlist.csv` as the identity universe and builds its first pass from:
  - exact local snapshots in `stats/manual/bballref_advanced.xlsx` and `C:\2k\epm.xlsx`
  - a live bball-index `LEBRON` family export in `stats/history/bball_index_impact.csv`
  - a current-season `EPM` overlay from `stats/history/dunksandthrees_epm.csv`
  - public `nbarapm.com` pulls in `stats/history/mamba.csv` and `stats/history/lebron.csv`
  - first-pass on-court proxies from `general_advanced.csv` and `tracking_defensive_impact.csv`
- Defense currently keeps `playerlist.csv` as the identity universe, enriches `RotationRole` from
  `stats/history/bball_index_defense.csv` when available, and builds its first pass from:
  - a live bball-index defense export in `stats/history/bball_index_defense.csv`
  - `stats/exports/impact_all_audit.csv -> Defensive Impact AggregateZ` as a direct carry-through for the workbook's `DEFENSIVE IMPACT`
  - public `nbarapm.com` pulls in `stats/history/nbarapm.csv` for rim, stop, steal, block, and forced-turnover rates
  - NBA.com history in `general_defense.csv`, `hustle.csv`, `defense_dashboard_3.csv`, and `defense_dashboard_l6ft.csv`
- Rebounding currently keeps `playerlist.csv` as the identity universe, enriches `RotationRole` from
  `stats/history/bball_index_rebounding.csv` when available, and builds its first pass from:
  - a live bball-index rebounding export in `stats/history/bball_index_rebounding.csv`
  - NBA.com history in `tracking_rebound.csv`, `general_traditional.csv`, `general_advanced.csv`, and `hustle.csv`
  - public `nbarapm.com` pulls in `stats/history/nbarapm.csv` for `SelfORebPct`
- `stats/scripts/pull_nbarapm_data.py` is now the active public JSON puller for `nbarapm.csv`, `mamba.csv`, and `lebron.csv`.
- `stats/scripts/pull_dunksandthrees_epm.py` is now the active public current-season `EPM` puller.
- `stats/scripts/pull_bball_index.py` now reads the active main `Metric` dropdown after opening it, which exposes the full
  live metric catalog from bball-index and strips the UI `*` suffixes from exported column names.
- The current Impact build now fills the workbook's core and extended `LEBRON` family slots, but it still leaves the
  RAPTOR / RPM / RAPM / DPM / DRIP / SPI families unmatched on purpose. The unmatched audit is the truth source for what
  still needs sourcing.
- Impact offense and defense currently renormalize their weighted groups across the groups that actually
  have data, so missing `Impact Luck` / RAPM-family buckets do not collapse every row to `-1`.
- `stats/history/lebron.csv` now serves as the safety fallback for core `LEBRON` rows that the live bball-index history
  export does not cover, especially before `2015-16`.
- Defense currently mirrors the workbook's special `Perimeter Defense` formula and also renormalizes across the pieces
  that actually have data, so missing older-source buckets do not collapse every row to `-1`.
- Rebounding currently mirrors the workbook's repeated defensive `Offensive Rebounding Chances Per 75 Possessions` slot
  and also renormalizes across the section buckets that actually have data, so older seasons without full bball-index,
  hustle, or nbarapm coverage do not collapse every row to `-1`.
- In `Playmaking -> Speed with Ball`, the `Ball Handle` slot comes from the playmaking
  `Ball Handle` aggregate z-score and the `Speed` slot now reads from the Physical
  category's `Speed AggregateZ` when `stats/exports/physical_all_audit.csv` exists.
  If that file is missing, the build falls back to `tracking_speed.csv -> AVG_SPEED`.
- Low-volume 3PT gate is set to `0.2`.
- Per-stat z-scores are clipped.
- Final aggregate category z-scores are not clipped before converting to ratings.
