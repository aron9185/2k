## Live Poll Rules

Live poll rows are now displayed only when still open.

Open means:

- `can_wager` is not false
- `is_locked` is not true
- `locks_at` is absent or later than current UTC time

This is enforced in two places:

- `recommend_live_polls.py` skips closed rows before scoring.
- `refresh_dashboard_data.py` filters stale CSV rows before rendering `live_polls.md`.

Live poll sort order:

1. `locks_at` / close time
2. Real app order through `feed_order`
3. sport / game / poll tie-breakers

`live_polls.py` now stores `feed_order` from the Real livefeed post index.

## Key Market Fixes

### PHI @ NYK NBA Total 215

Old behavior:

- only DK primary total `213.5` was available
- Real poll line `215` was evaluated as nearest-line fallback

New behavior:

- DK event `Alternate Total` ladder is ingested
- available ladder includes `212.5`, `213.5`, `214.5`, `215.5`, `216.5`
- row uses fitted line curve instead of nearest-line fallback

### PHI @ NYK Q1 Period Winner

Old behavior:

- no sportsbook match

New behavior:

- FanDuel 1st quarter tab is ingested
- Q1 spread proxy can recommend the period winner
- DK remains empty for some event subcategory pulls, so FanDuel is important here

### NBA Team FT%

Old behavior:

- team-stat polls could use sportsbook moneyline proxy

New behavior:

- NBA team-stat polls first try Real team compare stats
- uses team IDs, season, and season type from the game/poll payload
- postseason games request `seasonType=postseason`

Example:

```text
NYK 77.6% / PHI 80.1%
```

The FT% pick becomes PHI from official Real playoff compare stats.

## Pool Of The Day

Pool rows are modeled as a combined payout ladder:

- all child game-winner polls are scored
- payout tiers are read from the pool info when available
- expected return includes second-tier prizes where the pool pays for second most correct
- notes say whether cashing is more likely than losing

## Caveats

- Full live-poll refresh can take several minutes because it refreshes all sports and both books.
- FanDuel can intermittently fail during live fetches. Preserve existing rows if a one-off pull returns zero.
- DK's NBA quarter event subcategory is empty for some events.
- Past-game checks may not show in rendered markdown when `--not-started-only` is used.
- Large generated CSVs can dirty the repo again quickly.
