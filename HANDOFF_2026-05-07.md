### Live Polls

Live polls now filter out closed rows. A live poll must be:

- wagerable when `can_wager` is present
- not locked
- not past `locks_at`

The live poll renderer also filters stale CSV rows before display, so closed polls do not reappear if a refresh leaves old rows behind.

Live poll ordering:

1. close/lock time
2. Real livefeed order via `feed_order`
3. sport / game / poll tie-breakers

The current live poll dashboard showed `0 displayed polls` after filtering because all rows in the latest snapshot were already past lock time.

Relevant files:

- `real/live_polls.py`
- `real/recommend_live_polls.py`
- `real/refresh_dashboard_data.py`
- `real/output/dashboard/live_polls.md`

## Known Friction / Next Improvements

- Full live-poll refreshes can be slow because they refresh all sportsbook providers and sports.
- FanDuel can intermittently fail with SSL EOF or return zero rows on a live pull. Existing cached/previous rows are preserved when manually merging, but a future refactor should make provider refreshes more resilient by default.
- `--not-started-only` can hide rows for games that are already past relative to the local date/time. Use the CSV directly or omit the flag when checking a past-game fix.
- DK quarter markets are not always populated for every event. FanDuel quarter tabs are the current fallback for missing NBA 1Q lines.
- Generated sportsbook CSVs are large. Be intentional before committing them again.
